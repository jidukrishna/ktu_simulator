What is the WWW?

Definition The World Wide Web (WWW) is a service that operates over the Internet and provides access to linked multimedia documents (text, images, video, audio). It was invented by Tim Berners-Lee in 1989 at CERN so scientists could easily share information.

How the WWW Works (step by step)

  • You type a URL (e.g. https://www.example.com) into a browser.
  • The browser sends a request to a web server for that page.
  • The server responds with the web page (files).
  • The browser renders the content using HTML, CSS and JavaScript.
Fig 1.1 — Basic request/response flow between a browser and a web server

Browsers vs Servers

  • Web Browser (client) — software like Chrome/Firefox that requests and displays web pages.
  • Web Server — a computer/software that stores web pages and sends them to browsers on request.

URL and URI

Definition A URI (Uniform Resource Identifier) identifies a resource. It can be a URL (Uniform Resource Locator — locates the resource) or a URN (Uniform Resource Name). 
https://www.google.com/search?q=chatgpt

Domain Name System (DNS)

Definition DNS translates human-readable domain names (like www.google.com) into machine-readable IP addresses, so browsers can locate servers.

🏗️ Web Application Architecture 🔥 High-Yield
▾

Basic Structure of a Web Application

A web application has two broad sides that communicate over the network:

Client-Side (Frontend)
  • UI: what the user sees — built with HTML (structure), CSS (styling), JS (behavior)
  • Client-side logic: JavaScript handles input, basic processing, talks to the server
  • Example: a form validates input with JS before sending it to the server
Server-Side (Backend)
  • Application server: core logic engine — receives requests, processes rules, talks to the DB
  • Database: stores and manages all application data
  • Server-side logic: written in Python, Java, Ruby, PHP, Node.js — handles auth, validation, business rules
Fig 1.3 — Frontend, backend and database working together

Client–Server Model

Definition The Client is a device/application that initiates requests for services/resources. The Server is a computer/software that listens for client requests and responds accordingly.

How it works:

  • 1. Request — client sends a message asking for a resource/service.
  • 2. Response — server processes the request and sends data back.
  • 3. Example — a browser (client) requesting a webpage from a web server is the classic example.

Request–Response Cycle

  • The request carries a URL identifying the resource being asked for.
  • Routing directs the request to the correct handler based on the URL.
  • Middleware may execute functions that modify the request/response along the way.
  • After processing, the server sends back a response to the client.

Role of Databases and APIs

Definition An API (Application Programming Interface) defines how different software components interact — acting as an intermediary between client, server, or microservices. APIs can be web APIs (online interaction) or database APIs (direct DB access).
  • APIs let developers reuse existing code/functionality, saving time.
  • APIs enable integration of different software systems.
  • APIs abstract away the complexity of database interactions.
  • APIs and databases can be scaled independently to handle varying load.
Example When you log in to a site, the frontend sends your credentials via an API call; the backend validates them against the database and returns a response (success/failure) that the frontend then displays.

🎨 CSS3: Inline, Internal (Embedded), External Style Sheets
▾
Definition CSS3 (Cascading Style Sheets) lets you specify presentation (fonts, spacing, colors, positioning) separately from a document's structure/content. There are three levels, from lowest to highest priority: inline, internal (document-level), external.

1. Inline Styles

<h1 style="text-decoration:underline;color:blue;">M S Dhoni</h1>
<p style="color:red;font-size:32pt;">Indian cricketer.</p>
Rendered Output

M S Dhoni

Indian cricketer.

Syntax: style="property1:value1;property2:value2;". Applies only to that single element.

2. Internal / Embedded Style Sheets

<head>
  <style type="text/css">
    h1 {text-decoration:underline; color:red;}
    p  {color:green; font-weight:bold;}
  </style>
</head>
<body>
  <h1>Sundar Pichai</h1>
  <p>Indian-born American business executive.</p>
</body>
Rendered Output

Sundar Pichai

Indian-born American business executive.

Embedded inside the <head>. Applies only to the elements of that single document. MIME type is text/css.

3. External Style Sheets

<!-- in the HTML head -->
<link rel="stylesheet" type="text/css" href="ex.css"/>

/* ex.css */
h1 {text-decoration:underline; color:yellow;}
body {background-color:powderblue;}
p {color:blue;}
Rendered Output

External Style sheet

Styled entirely from ex.css — page background is powderblue.

  • A separate .css file reused across many pages/websites.
  • Gives an entire website a uniform look and feel.
  • Recommended for large projects — best maintainability.
Note The link element's rel attribute declares the linked document as a stylesheet; href gives its URL/path; type is text/css.

Box-Sizing Property 🔥

Rendered Output — same CSS, different box-sizing

visible width = 250px
border-box
visible width = 200px
content-box (default)
  • Width/height style only the content area
  • Padding & border are added on top, expanding the box
  • Formula: Total Width = Width + Left/Right Padding + Left/Right Border
  • Example: width:200px; padding:20px; border:5px → total = 250px
border-box
  • Width/height represent the entire visible box
  • Padding & border shrink the inner content instead of expanding the box
  • Formula: Total Width = Specified Width (content shrinks automatically)
  • Example: width:200px; padding:20px; border:5px → total stays 200px, content shrinks to 150px
Common Mistake Forgetting that the default box model is content-box — this is why layouts "grow" unexpectedly when padding/border is added, unless box-sizing:border-box is set.
📐 CSS Flexbox 🔥 High-Yield
▾
Definition Flexbox (Flexible Box Layout) is a one-dimensional layout model used to align, distribute, and organize elements — laying items out as a row OR a column at a time. 
.container {
  display: flex;               /* makes this a flex container */
  flex-direction: row;         /* row | row-reverse | column | column-reverse */

  align-items: center;         /* aligns items on the cross axis */

.item {
  flex-grow: 1;    /* how much an item expands into free space */
  flex-shrink: 1;  /* how much an item shrinks to avoid overflow */
  flex-basis: 100px; /* initial size before space distribution */
  order: 2;        /* re-orders visually without changing HTML */
}

Definition A media query applies a block of CSS only when device conditions (viewport width, orientation, resolution) are met — the core building block of responsive design.

⚡ JavaScript Basics: Syntax, Variables (var/let/const) 🔥 High-Yield
▾
Definition JavaScript is a client-side scripting language used to enhance the functionality and appearance of web pages — improve design, validate forms, detect browsers, create cookies, etc. Java is strongly typed; JavaScript is dynamically typed.

Where can JS be placed?

  • Inline JavaScript — directly on an element as an event attribute, e.g. onclick.
  • Internal JavaScript — inside <script></script> tags in the HTML file (commonly in <head>).
  • External JavaScript — a separate .js file linked in, reusable across multiple pages, easier to maintain.
<script>
  document.write("Hello World!");
</script>
Rendered Output
Hello World!

  • Variables must have a name and value; declared before use.
  • Variable names are case sensitive and must begin with a letter or underscore _.
  • An identifier is letters, digits, underscores, and $ that doesn't start with a digit and isn't a reserved keyword.
Common Mistake Using var inside loops/blocks expecting block scoping — var leaks outside the block, which can cause bugs. Prefer let/const.

Primitive vs Non-Primitive 🔥

Primitive — Immutable
var s = 'This is a string.';
s = 'H';
alert(s); // 'H' (s reassigned, original unchanged in memory)
Non-Primitive — Mutable
var arr = ['one','two','three'];
arr[1] = 'TWO';
alert(arr); // ['one','TWO','three'] — modified in place

Details

  • Number — all numeric values stored as double-precision floating point (e.g. 72, 7.2, .72, 7e2).
  • String — delimited by ' or "; can include escape sequences like \t.
  • Boolean — true/false, used in conditions.
  • Null — means "no value"; used when a variable is explicitly set to have no value.
  • Undefined — a variable has been declared but not assigned a value, e.g. var carName;

Arrays & Objects

// Array — square brackets, comma-separated items
const cars = ["Saab", "Volvo", "BMW"];

// Object — curly braces, name:value pairs
const person = {firstName:"John", lastName:"Doe", age:50, eyeColor:"blue"};
console.log(cars);
console.log(person.firstName, person.age);
Console Output
['Saab', 'Volvo', 'BMW']
John 50
Note JavaScript arrays are dynamic and heterogeneous — they can change size after creation and hold mixed types.

▾
Definition TypeScript is a syntactic superset of JavaScript that adds static typing. JavaScript is loosely typed (hard to know what data types are passed around); TypeScript lets you declare types and reports errors when types don't match.

Simple / Primitive Types

TypeExample

Explicit vs Implicit Typing

// Explicit — you write the type out
let firstName: string = "Dylan";

// Implicit — TypeScript infers the type from the value
let firstName = "Dylan";   // inferred as string

Array, Tuple, Union & Literal Types

let numbers: number[] = [1, 2, 3];
let names: string[] = ["Alice", "Bob"];

// Tuple — fixed-length array with specific types per slot

// Union — allows more than one type
let id: number | string = 101;
id = "A123";

// Literal type — only specific values allowed
let direction: "up" | "down";
direction = "up";

Worked Example: Student Report

interface Student {
  name: string;
  rollNo: number;
  marks: number[];
  isPassed: boolean;
}

const student1: Student = {
  name: "Shini", rollNo: 101, marks: [85, 90, 78], isPassed: true
};

function displayStudentInfo(student: Student): string {
  const average = student.marks.reduce((sum, m) => sum + m, 0) / student.marks.length;
  return `Name: ${student.name}, Average: ${average.toFixed(2)},
  Status: ${student.isPassed ? "Passed" : "Failed"}`;
}
console.log(displayStudentInfo(student1));

Name: Shini, Average: 84.33,
Status: Passed
